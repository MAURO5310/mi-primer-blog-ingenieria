GIT

instalar GIT

Para comenzar debemos instalar en nuestro computador el sistema GIT Para esto debemos ingresar a nuestro buscador el siguiente link https://git-scm.com/ y descargar segun el sistema operativo de nuestro computador

Crear un nuevo directorio o carpeta

Si quieres crear un nuevo proyecto, primero es ejecutar los comandos

ls.

para ubicar donde crear nuestra nueva carpeta y después ejecutar comando

cd.

para ingresar al lugar donde crearemos una carpeta y ejecutamos los comandos

mkdir nombre-del-proyecto.

(Creación Git) en nuestra terminal para crear la carpeta

Inicializar el repositorio

Ahora inicializa un repositorio Git en el directorio actual:

git init:

Crea un nuevo repositorio vacío de Git. Esto crea un subdirectorio llamado

git

, donde Git almacena toda la información de versión y metadatos.

Agregar archivos

Crea algunos archivos en tu carpeta. Por ejemplo, puedes crear un archivo html:

code index.html

Crea un archivo vacío llamado "index.html">

Ahora se puede trabajar en el documento de index.html

Después ejecutamos comando

git status

para mirar el status del repositorio que estamos trabajando

Aparece el archivo index.html en color rojo, lo que significa que ese archivo no ha sido rastreado por lo que no realizará ninguna copia del archivo (index.html) hasta que sea rastreado, para lo cual ejecutamos el comando

git add

(index.html) y nuevamente

git status

lo que arrojara que hay un nuevo archivo que se llama index.html

CREACION DE BRANCHESA

Guardar cambios: git commit

Una vez que los archivos están en la staging, debes confirmarlos con

git commit

Esto guarda una versión permanente de los cambios en el historial de Git.

git commit m

"Mensaje que describe los cambios"

Branches (Ramas) y Cambiar entre ellas

Git permite trabajar en diferentes ramas para mantener un desarrollo organizado, facilitando la creación de características nuevas o corrección de errores sin afectar el código principal.

git checkout -b Verificacion_Git

Cuando terminas de trabajar en una rama, puedes fusionarla con otra (normalmente con la rama principal main o master). Por ejemplo, cambiamos a la rama main

git checkout main

y una vez allí podemos fusionar el contenido de alguna de las otras ramas:

git merge nombre-de-la-rama

Esto combinará los cambios de la nombre-de-la-rama en la rama actual. Recuerda que puedes usar

git status

para ver el estado actual de tu repositorio, mostrándote qué archivos han sido modificados, cuáles están listos para ser confirmados (en el área de staging) y cuáles no se están rastreando aún.

Agregar un repositorio remoto

Crea un repositorio remoto en una plataforma: Si aún no tienes un repositorio remoto, primero debes crearlo en una plataforma como GitHub, GitLab, o Bitbucket. Una vez que lo crees, obtendrás la URL del repositorio remoto, que puede ser algo como https://github.com/usuario/repositorio.git o git@github.com:usuario/repositorio.git. Configura el repositorio remoto en tu proyecto local: Para agregar el repositorio remoto a tu proyecto local, usa el comando:

git remote add origin

Verificar que se agregó el repositorio remoto

Puedes verificar si el repositorio remoto fue agregado correctamente con este comando:

git remote -v
